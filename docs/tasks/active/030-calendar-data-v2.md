# 030 캘린더 데이터와 동기화 v2 기반

## 상태
- 상태: 활성
- 브랜치: `feat/calendar-data-v2`

## 목표
일정, 기념일, 색상, 날짜 셀 장식, 반복과 알림 설정을 로컬과 서버에서 안전하게
저장·동기화할 수 있는 하위 호환 데이터 기반을 만든다.

## 의존성
- 작업 000 완료
- 작업 순서상 020 병합 후 시작

## 범위
- `CalendarItem`, `ICalendarRepository`, 일정·반복·알림 관련 타입을 추가한다.
- `DateCellDecoration`과 `SyncState`를 추가한다.
- 다중 SQL 마이그레이션 실행기와 v2 서버 스키마·API·통합 커서를 구현한다.
- v1 API를 유지하고 기존 `todos.json`을 버전이 있는 새 로컬 문서로 가져온다.

## 제외 범위
- 발생 일정 계산, 편집 UI와 알림 창은 구현하지 않는다.
- v1 API와 기존 로컬 파일을 삭제하지 않는다.

## 설계 결정
- 일정 시리즈는 한 행으로 저장하고 날짜 장식은 별도 동기화 엔터티로 저장한다.
- v2 동기화 변경은 엔터티 유형, payload와 cursor를 포함한다.
- 가져오기는 원자적으로 수행하며 성공 후에도 기존 파일은 보존한다.

## 완료 조건
- 기존 사용자 데이터가 손실 없이 새 저장소에서 조회된다.
- 일정과 날짜 장식의 CRUD·삭제 tombstone·커서 동기화가 동작한다.
- v1 클라이언트 계약이 유지된다.

## 검증
- 로컬 파일 가져오기·실패 복구·계정 분리 테스트를 추가한다.
- 서버 마이그레이션 순서, v1 호환, v2 CRUD와 커서 테스트를 추가한다.
- .NET 및 서버 전체 테스트와 빌드를 실행한다.
