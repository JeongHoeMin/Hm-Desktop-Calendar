# 150 설정 창 셸과 트레이 메뉴 확장

## 상태

- 상태: 활성
- 브랜치: `feat/settings-window-and-tray`

## 목표

설정 항목을 담을 설정 창을 만들고, "종료"뿐인 트레이 메뉴를 실사용 가능한
진입점으로 확장한다. 이후 설정 작업(160, 170, 180, 250)은 이 창에 섹션만
추가하면 된다.

## 의존성

- 140 (앱 설정 저장소 확장)

## 범위

- `SettingsWindow` + `SettingsViewModel`을 추가한다. 단일 인스턴스 패턴은
  `App.axaml.cs`의 일정 모아보기 창(`ShowScheduleOverview`) 패턴을
  재사용한다.
- 트레이 메뉴(`App.axaml.cs`의 `CreateTrayIcon`)를 확장한다: 설정, 일정
  모아보기, 위치 편집, 로그인/로그아웃(세션 상태 반영), 종료.
- 달력 창 ☰ 메뉴(`MainWindow.ShowMenu`)에 "설정" 항목을 추가한다.
- 초기 설정 창 내용: 앱 버전 표시, 창 위치 초기화 버튼(저장된 위치 삭제 후
  기본 위치 적용).

## 제외 범위

- 개별 설정 항목(160, 170, 180, 250에서 섹션 추가).
- 트레이 아이콘 이미지 교체(현재 avalonia-logo.ico) — 필요 시 별도 작업.

## 설계 결정

- 트레이 메뉴의 로그인/로그아웃 항목은 `AuthSession`의 상태 변경 이벤트로
  `NativeMenuItem` 표시를 갱신한다.
- 설정 창은 즉시 적용 방식이다(확인/취소 버튼 없음). 각 항목이 변경 즉시
  설정 저장소에 저장한다.
- 설정 파일에는 이후 작업의 다른 항목도 함께 저장되므로 위치 초기화 시 파일 전체를
  삭제하지 않고 기본 Bounds `(100, 100, 980, 680)`를 즉시 적용·저장한다.

## 완료 조건

- 트레이와 ☰ 메뉴 양쪽에서 설정 창이 열리고 중복 인스턴스가 생기지 않는다.
- 트레이에서 일정 모아보기, 위치 편집, 로그인/로그아웃, 종료가 동작한다.
- 창 위치 초기화가 즉시 반영되고 재시작 후에도 유지된다.

## 검증

- 뷰모델 단위 테스트(위치 초기화, 세션 상태에 따른 메뉴 문구).
- 트레이 UI는 자동화가 불가하므로 수동 확인 항목을 PR에 기록한다.
- UI 변경이므로 PR에 스크린샷을 포함한다.
