# 110 비활성 작업 디렉터리 복구

## 상태

- 상태: 활성
- 브랜치: `docs/restore-inactive-task-directory`

## 목표

대기 작업이 없을 때도 `docs/tasks/inactive` 상태 디렉터리를 저장소에 유지해 새 작업을
정책에 따라 등록하고 활성화할 수 있게 한다.

## 의존성

- 작업 100과 PR #13이 `main`에 병합돼 있어야 한다.
- 현재 열린 PR과 GitHub 이슈가 없어야 한다.

## 범위

- `docs/tasks/inactive` 디렉터리를 빈 상태에서도 추적할 수 있도록 `.gitkeep`을 추가한다.
- 작업 관리 구조 복구 과정과 검증 결과를 이 문서에 기록한다.

## 제외 범위

- 애플리케이션과 서버 코드를 변경하지 않는다.
- 새 제품 기능이나 후속 구현 작업을 임의로 등록하지 않는다.
- 작업 관리 정책 자체를 변경하지 않는다.

## 설계 결정

- 기존 `active`와 `completed` 디렉터리의 방식에 맞춰 빈 상태 디렉터리를 `.gitkeep`으로
  보존한다.
- 다음 제품 작업은 사용자 목표가 확정된 뒤 별도 작업 문서와 PR로 진행한다.

## 완료 조건

- 저장소를 새로 체크아웃해도 `docs/tasks/inactive` 디렉터리가 존재한다.
- 활성 작업 문서가 이 문서 하나뿐이다.
- 제품 코드와 기존 완료 작업 문서에는 변경이 없다.

## 검증

- 세 상태 디렉터리와 활성 작업 문서 개수를 확인한다.
- `git diff --check`와 전체 diff를 검토한다.
