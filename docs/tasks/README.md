# 작업 관리 안내

이 디렉터리는 하네스 엔지니어링 정책에 따라 저장소의 작업 상태와 구현 결정을
관리한다. 작업별 요구사항과 진행 정보는 제품 개요나 `AGENTS.md`가 아니라 이곳에
기록한다.

## 상태 디렉터리

- `inactive`: 구현 순서를 기다리는 승인된 작업
- `active`: 현재 구현 또는 리뷰 수정 중인 단 하나의 작업
- `completed`: 구현과 필수 검증을 마치고 PR을 연 작업

동시에 두 개 이상의 작업 문서를 `active`에 두지 않는다. PR 리뷰에서 수정 요청을
받으면 해당 문서를 `completed`에서 `active`로 되돌린다.

## 작업 절차

1. 선행 PR이 병합됐는지 확인하고 최신 `main`을 받는다.
2. 다음 작업 문서를 `inactive`에서 `active`로 이동한다.
3. 문서의 결정과 완료 조건을 현재 저장소 상태에 맞게 보완한다.
4. 전용 브랜치에서 범위를 벗어나지 않게 구현하고 검증 결과를 기록한다.
5. 커밋과 PR을 만든 뒤 문서에 해당 정보를 기록한다.
6. 작업 문서를 `completed`로 이동하고 같은 PR에 반영한다.
7. PR URL을 사용자에게 전달하고 병합 확인 전까지 다음 작업을 시작하지 않는다.

## 문서 규칙

- 파일명은 `세 자리 ID-짧은 영문 식별자.md` 형식을 사용한다.
- 모든 설명은 한국어로 작성한다. 코드 식별자와 경로는 원문을 유지할 수 있다.
- 작업 하나는 독립적으로 검토하고 병합할 수 있는 PR 하나에 대응한다.
- 범위를 추가해야 하면 현재 작업에 조용히 포함하지 말고 별도 비활성 작업을 만든다.
- 프로그램 목적이나 현재 시스템 경계 확인이 필요할 때만
  `docs/product-overview.md`를 참고한다.

## 필수 항목

모든 작업 문서는 ID, 상태, 목표, 의존성, 범위, 제외 범위, 설계 결정, 완료 조건,
검증, 커밋과 PR URL을 포함해야 한다. 새 문서는 `_template.md`를 복사해 작성한다.
