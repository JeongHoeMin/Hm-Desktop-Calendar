# 080 애플리케이션 자체 알림 팝업

## 상태
- 상태: 비활성
- 브랜치: `feat/reminder-popup`

## 목표
Windows 알림센터를 사용하지 않고 일정 시각에 제목과 메모를 보여주는 앱 자체
알림 팝업과 안정적인 예약 엔진을 제공한다.

## 의존성
- 작업 030, 040, 050, 070 완료

## 범위
- 가짜 시계로 테스트 가능한 `ReminderScheduler`를 구현한다.
- 정시, 5·15·30분, 1시간, 1일 전과 사용자 지정 오프셋을 지원한다.
- 시간이 없는 일정은 별도 알림 시각을 필수로 입력한다.
- 팝업에 확인, 5·10·30분 미루기와 편집 열기를 제공한다.
- 시작 시 최근 24시간 내 놓친 알림만 한 번 복구한다.

## 제외 범위
- Windows 알림센터, 서버 푸시와 앱 종료 중 실시간 팝업은 지원하지 않는다.
- 확인·미루기 상태를 계정 간 동기화하지 않는다.

## 설계 결정
- 알림 설정은 동기화하고 표시·확인·미루기 기록은 장치별 로컬 데이터로 둔다.
- 여러 알림이 동시에 도래하면 중복 창을 만들지 않고 순서대로 표시한다.
- 일정 시간은 실행 장치의 로컬 시간으로 해석한다.

## 완료 조건
- 각 오프셋, 반복 발생 일정과 누락 복구에서 중복 없이 팝업이 열린다.
- 팝업에 메모가 표시되고 미루기와 편집 열기가 동작한다.

## 검증
- 정시·오프셋·무시간·누락·중복·미루기·종료 테스트를 가짜 시계로 추가한다.
- 앱 종료 정리와 여러 알림 큐를 수동 확인한다.
