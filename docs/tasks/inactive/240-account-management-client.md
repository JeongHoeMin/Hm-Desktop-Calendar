# 240 클라이언트 계정 관리 UI

## 상태

- 상태: 비활성
- 브랜치: 미정

## 목표

로그인한 사용자가 앱 안에서 비밀번호 변경과 계정 삭제를 수행할 수 있게
한다.

## 의존성

- 230 (계정 관리 API가 `main`에 병합되고 서버에 배포 가능해야 한다)

## 범위

- `Authentication/AuthSession.cs`에 `ChangePasswordAsync`와
  `DeleteAccountAsync`를 추가한다.
- ☰ 메뉴의 로그인 상태에 "계정" 항목을 추가하고 계정 창을 연다.
  - 비밀번호 변경 폼: 현재/신규/확인 입력.
  - 계정 삭제: 확인 문구 입력 후에만 버튼 활성화.
- 변경 성공 시 세션을 정리하고 재로그인을 안내한다.
- 삭제 성공 시 로그아웃하고 로컬 계정 스코프 데이터 삭제 여부를 선택받는다.

## 제외 범위

- 서버 변경(230에서 완료).
- 이메일 변경.

## 설계 결정

- 계정 창은 `LoginWindow`와 같은 단일 인스턴스 패턴을 따른다.
- 서버 오류는 한국어 메시지로 매핑한다(예: 401 → "현재 비밀번호가 일치하지
  않습니다").
- 삭제 확인 문구는 계정 이메일 입력으로 한다.

## 완료 조건

- 비밀번호 변경 → 재로그인 → 동기화가 정상 동작한다.
- 계정 삭제 후 다른 기기의 세션도 다음 토큰 갱신 시점에 로그아웃된다.

## 검증

- 뷰모델 단위 테스트(입력 검증, 확인 문구 활성화 규칙, 오류 매핑).
- 로컬 서버를 대상으로 변경·삭제 수동 시나리오를 수행하고 PR에 기록한다.
- UI 변경이므로 PR에 스크린샷을 포함한다.
