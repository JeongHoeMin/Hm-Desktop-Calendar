# 020 달력 상단과 메뉴·날짜 이동 개선

## 상태
- 상태: 완료
- 브랜치: `feat/calendar-shell-navigation`

## 목표
달력 상단 정보 구조, 메뉴 닫힘 동작, 연도·월 이동과 최근 동기화 상태를 예측
가능한 방식으로 개선한다.

## 의존성
- 작업 010 완료

## 범위
- 상단 좌측을 메뉴, 이전, 연도, 월, 다음 순으로 묶는다.
- 우측에 마지막 성공 동기화 시각과 진행·실패·로컬 전용 상태를 표시한다.
- 연도 목록, 12개월 그리드와 오늘 바로가기를 제공한다.
- 메뉴를 단일 재사용 Flyout으로 만들고 외부 클릭과 메뉴 버튼 재클릭으로 닫는다.

## 제외 범위
- 서버 동기화 프로토콜과 일정 데이터 모델은 바꾸지 않는다.
- 일정 모아보기 메뉴 항목은 후속 작업에서 추가한다.

## 설계 결정
- 동기화 표시는 성공 시각을 기준으로 하고 실패해도 마지막 성공 시각을 보존한다.
- 동기화 저장소가 진행·성공·실패 상태 이벤트를 발행하고 달력 ViewModel이 사용자용
  상태 문구로 변환한다. 로그인하지 않은 상태는 로컬 전용으로 표시한다.
- 성공 시각은 현재 실행 세션의 마지막 성공 시각이며 로컬 시간으로 표시한다.
- 연도 목록은 표시 연도를 중심으로 앞뒤 10년을 제공한다.
- 연도나 월 선택은 표시 월만 이동하며 일정 편집 창을 자동으로 열지 않는다.
- 메뉴와 날짜 선택기는 창마다 하나의 Flyout 인스턴스를 재사용한다.

## 완료 조건
- 헤더가 지정된 좌우 구조를 가지며 모든 상태 문구가 올바르게 갱신된다.
- 메뉴와 날짜 선택 Flyout이 재클릭·외부 클릭·선택 완료에서 정상 종료된다.

## 검증
- Release 빌드를 경고 0개, 오류 0개로 통과했다.
- 기존 회귀 테스트 11개와 ViewModel 날짜 이동·동기화 상태 테스트 2개를 통과했다.
- 실제 데스크톱 창에서 연도·월·메뉴 Flyout의 재클릭, 외부 클릭, 항목 선택 종료와
  월 선택 결과를 확인했다.
- 1080×969 데스크톱 배치와 700×480 전경 편집 모드에서 렌더링을 검토했다.
- 기본·우측·음수 좌표 세로 모니터의 3대 토폴로지를 확인하고, 기존 음수 좌표 및
  Bounds 회귀 테스트로 다중 모니터 호환성을 확인했다.
