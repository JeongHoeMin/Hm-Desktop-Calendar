# 230 계정 관리 API

## 상태

- 상태: 완료
- 브랜치: `feat/account-management-server`

## 목표

비밀번호 변경과 계정 삭제 API를 추가해 계정 수명주기의 최소 요건을 갖춘다.
이메일 인프라 없이 가능한 범위만 다룬다.

## 의존성

- 120 (서버 비밀 정보 정리)
- 210 선행 권장(민감 엔드포인트에 강화 rate-limit 적용)

## 범위

- `POST /v1/auth/password`: 액세스 토큰과 현재 비밀번호를 검증한 뒤
  argon2id로 재해싱하고, 해당 사용자의 refresh 토큰을 전부 폐기해
  재로그인을 강제한다.
- `DELETE /v1/auth/me`: 액세스 토큰과 비밀번호를 검증한 뒤 사용자와 일정,
  날짜 장식, 레거시 todo, refresh 토큰을 삭제한다. FK cascade를 확인하고
  부족하면 `003_*.sql` 마이그레이션으로 보강한다.
- 두 엔드포인트에 강화 rate-limit을 적용한다(210 도입 시).

## 제외 범위

- 비밀번호 재설정과 이메일 인증. SMTP·메일러 인프라가 없고, 로컬 우선 제품
  경계상 서버 계정은 선택 기능이므로 운영 메일 인프라가 준비되는 시점에
  별도 작업으로 등록한다.
- 이메일 변경, 세션 목록 조회.
- 클라이언트 UI(작업 240).

## 설계 결정

- 삭제는 물리 삭제로 한다(soft delete 없음 — 개인 데스크톱 제품 특성).
- 실패 응답은 기존 auth 오류 포맷을 재사용하고, 비밀번호 불일치는 401을
  반환한다.
- 비밀번호 변경 후 기존 액세스 토큰은 남은 TTL(15분) 동안 유효함을 문서에
  명시한다(refresh 폐기로 갱신은 차단됨).
- 기존 `001_initial.sql`과 `002_calendar_v2.sql`의 사용자 종속 FK가 모두
  `ON DELETE CASCADE`이므로 추가 마이그레이션 없이 사용자 행 삭제에 맡긴다.

## 완료 조건

- 비밀번호 변경 후 기존 refresh 토큰으로 갱신이 실패한다.
- 계정 삭제 후 로그인과 데이터 조회가 불가하고 관련 행이 남지 않는다.

## 검증

- `pnpm test`에 라우트 테스트를 추가한다: 성공 경로, 비밀번호 불일치,
  refresh 토큰 폐기, 삭제 cascade.
- 마이그레이션 추가 시 기존 데이터에 대한 적용을 검증한다.

### 현재 검증 결과

- `pnpm build`: 성공.
- `pnpm test`: 7개 파일, 27개 테스트 통과. 비밀번호 변경 성공·불일치, 모든 갱신
  세션 폐기, 계정 삭제 성공·불일치, FK cascade 선언과 강화 rate-limit을 검사한다.
- WSL Docker Compose의 빈 PostgreSQL 16에서 계정 등록(201), 비밀번호 변경(200),
  기존 refresh 실패(401), 기존 비밀번호 로그인 실패(401), 새 비밀번호 로그인(200),
  계정 삭제(204), 삭제 후 로그인 실패(401)를 확인했다.
- 삭제 전에 refresh 세션, 레거시 todo, 일정, 날짜 장식과 동기화 변경을 만들었고
  삭제 후 사용자와 각 종속 테이블의 해당 사용자 행이 모두 0건임을 확인했다.
- 검증 후 Compose 컨테이너, 네트워크와 테스트 PostgreSQL 볼륨을 삭제했다.
