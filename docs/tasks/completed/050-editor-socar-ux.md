# 050 일정 편집 UI·UX와 텍스트 색상·메모

## 상태

- 상태: 완료
- 브랜치: `feat/editor-socar-ux`

## 목표

일정 추가와 수정을 하나의 쏘카프레임 기반 폼으로 통합하고 메모와 일정 텍스트
색상을 명확하고 안전하게 편집할 수 있게 한다.

## 의존성

- 작업 010, 030, 040 완료

## 범위

- 추가·수정 공용 폼과 일정 카드 목록 구조로 편집 창을 재설계한다.
- 제목, 시각, 메모와 완료 상태를 일관된 저장 흐름으로 편집한다.
- 접근성 팔레트와 사용자 지정 HEX 텍스트 색상 입력을 제공한다.
- 저장되지 않은 변경, 유효성 오류와 삭제 확인을 처리한다.
- 편집기와 달력 미리보기를 v2 `ICalendarRepository`와 발생 일정 조회에 연결한다.
- 기존 계정 범위, 통합 커서 동기화 상태와 로컬 우선 동작을 v2 저장 흐름에서 유지한다.

## 제외 범위

- 날짜 셀 배경색, 기념일, 반복 규칙 UI와 알림 팝업은 후속 작업으로 둔다.
- 날짜별 발생 인스턴스 편집은 지원하지 않고 기존 시리즈 규칙을 보존한다.

## 설계 결정

- 추가와 수정은 동일한 편집 초안 ViewModel과 검증 규칙을 사용한다.
- 텍스트 색상은 `#RRGGBB` 형식으로 정규화하고 중립 일정 칩 `#F8F9FC`에서
  WCAG AA 일반 텍스트 기준인 4.5:1 이상을 만족해야 저장할 수 있다.
- 접근성 팔레트 선택과 사용자 지정 HEX 입력은 같은 색상 검증 경로를 사용한다.
- 이전 v2 기본색 `#3B82F6`은 읽을 때 접근 가능한 `#0041E6`으로 변환한다.
- 메모는 접힌 보조 정보가 아니라 주요 편집 필드로 표시한다.
- 다른 일정·날짜 선택과 창 닫기는 저장되지 않은 변경을 확인한 뒤 진행한다.
- 삭제는 명시적인 확인 뒤 원본 일정 ID에 적용한다.
- 편집하지 않는 기간·반복·알림 필드는 원본에서 그대로 보존한다.
- 달력 미리보기는 발생 일정의 원본 텍스트 색상을 사용한다.

## 완료 조건

- 추가·수정·삭제·완료와 취소 흐름에서 데이터 손실이나 중복 저장이 없다.
- 팔레트와 유효한 사용자 색상이 달력 미리보기까지 반영된다.
