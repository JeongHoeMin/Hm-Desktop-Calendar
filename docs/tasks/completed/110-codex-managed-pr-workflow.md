# 110 Codex 주도 PR 처리 정책

## 상태

- 상태: 완료
- 브랜치: `docs/codex-managed-pr-workflow`

## 목표

Codex가 구현과 검증 이후 PR 생성에 그치지 않고 정상적인 병합 조건을 확인해 병합과
기본 브랜치 반영 확인까지 책임지도록 저장소 개발 정책을 변경한다.

## 의존성

- 작업 100과 PR #13이 `main`에 병합돼 있어야 한다.
- 저장소의 필수 검사, 요구된 리뷰와 브랜치 보호 규칙을 `gh`로 확인할 수 있어야 한다.

## 범위

- Codex가 별도의 사용자 병합 지시를 기다리지 않고 PR을 정상 병합하도록 작업 절차를
  변경한다.
- 충돌, 실패한 검사와 누락된 필수 리뷰가 있을 때의 중단 및 보고 동작을 명시한다.
- 완료 정의를 PR 병합과 기본 브랜치 반영 확인까지 확장한다.

## 제외 범위

- Codex의 자기 승인, 브랜치 보호 규칙 우회, 관리자 강제 병합과 강제 푸시를 허용하지
  않는다.
- GitHub 저장소의 브랜치 보호 설정이나 필수 검사 구성을 변경하지 않는다.
- 애플리케이션과 서버 코드를 변경하지 않는다.

## 설계 결정

- Codex가 만든 PR은 정상 병합 조건이 충족되면 Codex가 기본적으로 병합한다.
- 사용자 확인은 매 PR의 정상 병합 조건으로 요구하지 않는다.
- 외부 조건으로 정상 병합할 수 없을 때만 원인을 사용자에게 보고하고 멈춘다.
- 병합 후 원격 PR 상태와 기본 브랜치 반영을 확인해야 다음 작업을 시작할 수 있다.

## 완료 조건

- 작업 절차와 GitHub 정책에서 사용자별도 병합 지시 요구가 제거된다.
- Codex의 기본 병합 책임과 금지 동작이 모호하지 않게 구분된다.
- 완료 정의가 PR 병합과 기본 브랜치 반영 확인을 포함한다.

## 검증

- 변경된 정책 문구와 전체 diff를 검토한다.
- 사용자 지시가 없어도 정상 PR을 병합하는 흐름과 차단 조건을 대조한다.
- `git diff --check`로 공백 오류가 없는지 확인한다.
