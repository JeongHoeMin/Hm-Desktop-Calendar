# 100 에이전트 PR 병합 정책

## 상태

- 상태: 완료
- 브랜치: `docs/agent-pr-merge-policy`

## 목표

사용자가 현재 요청에서 명시적으로 지시한 경우 Codex가 필수 조건을 확인하고 PR을
직접 병합할 수 있도록 저장소 개발 정책을 명확히 한다.

## 의존성

- 작업 090과 PR #12 병합 완료

## 범위

- 하네스 엔지니어링 작업 절차에 명시적 병합 요청 처리 순서를 추가한다.
- GitHub 정책에 병합 전 검사와 금지되는 우회 동작을 명시한다.

## 제외 범위

- 자동 병합, 사용자 요청 없는 병합, 자체 승인과 브랜치 보호 규칙 우회는 허용하지
  않는다.
- GitHub 저장소의 브랜치 보호 설정 자체는 변경하지 않는다.

## 설계 결정

- 병합 권한은 현재 사용자 요청의 명시적인 지시에만 근거한다.
- 병합 전에 초안 여부, 병합 가능 상태, 필수 검사와 요구된 리뷰를 확인한다.
- 필수 조건이 충족되지 않으면 병합하지 않고 정확한 차단 원인을 보고한다.
- 병합 후 원격 PR 상태와 기본 브랜치 반영을 확인한다.

## 완료 조건

- 정책 문서가 허용 조건과 금지 조건을 모호하지 않게 구분한다.
- 사용자의 명시적 지시가 없는 기존 기본 동작은 사용자 검토 대기로 유지된다.

## 검증

- Markdown 내용과 전체 diff를 검토한다.
- `git diff --check`로 공백 오류가 없는지 확인한다.
