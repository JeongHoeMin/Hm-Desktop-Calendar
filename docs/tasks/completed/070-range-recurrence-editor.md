# 070 기간·반복 일정 편집

## 상태
- 상태: 완료
- 브랜치: `feat/range-recurrence-editor`

## 목표
사용자가 단일, 연속 기간 또는 표준 반복 규칙을 편집 창에서 설정하고 어느
날짜에서 열어도 전체 시리즈를 일관되게 수정할 수 있게 한다.

## 의존성
- 작업 030, 040, 050 완료

## 범위
- 단일·기간·반복 모드 선택과 모드별 입력을 제공한다.
- 종료일, 반복 빈도, 간격, 주간 요일과 반복 종료일·무기한을 편집한다.
- 파생 날짜 항목에서 같은 시리즈 원본 편집 창을 연다.
- 전체 시리즈 수정·완료·삭제임을 UI에 명시한다.

## 제외 범위
- 한 날짜만 수정, 날짜별 완료, 기간과 반복 결합, 반복 횟수는 지원하지 않는다.

## 설계 결정
- 모드 변경 시 사용하지 않는 필드는 명시적으로 초기화한다.
- 잘못된 종료일, 빈 요일과 1 미만 간격은 저장 전에 차단한다.
- 단일·기간·반복 모드는 각각 종료일과 반복 규칙을 배타적으로 소유하며 기간
  모드는 시작일보다 늦은 종료일을 요구한다.
- 반복 모드는 매일·매주·매월·매년 빈도, 정수 간격, 매주 선택 요일과 선택적인
  포함 종료일을 편집한다. 반복 횟수는 노출하지 않는다.
- 기념일은 작업 060의 계약대로 무기한 매년 반복에 고정하고 일반 일정의 반복
  모드 입력과 섞지 않는다.
- 파생 날짜에서 조회된 `CalendarOccurrence.Item`은 시리즈 원본이므로 기존 ID와
  시작일을 유지해 전체 시리즈 수정·완료·삭제에 사용한다.

## 완료 조건
- 모든 지원 규칙을 UI에서 만들고 다시 열어 손실 없이 수정할 수 있다.
- 어느 발생 날짜에서 수정해도 전체 시리즈가 즉시 갱신·동기화된다.

## 검증
- 모드 전환·유효성 검사·직렬화와 전체 시리즈 편집 테스트를 추가한다.
- 월 경계와 여러 발생 날짜에서 편집·완료·삭제를 수동 확인한다.

- .NET Release 빌드를 경고 0개, 오류 0개로 통과했다.
- 모드 전환 초기화, 종료일·간격·요일 검증, 네 반복 빈도의 재편집, 원격 직렬화,
  월 경계 기간과 파생 날짜 전체 시리즈 수정·완료·삭제를 포함한 회귀 테스트
  43개를 통과했다.
- 서버 테스트 9개와 TypeScript 빌드를 통과했다.
- Avalonia Headless로 640×560 최소 크기에서 격주 요일·간격·종료일과 스크롤
  접근성을 확인하고, 820×700에서 월 경계 기간의 전체 날짜와 시리즈 안내를
  확인했다.
- 검증 캡처는 `docs/screenshots/070-editor-weekly-minimum.png`와
  `docs/screenshots/070-editor-range-month-boundary.png`에 기록했다.
