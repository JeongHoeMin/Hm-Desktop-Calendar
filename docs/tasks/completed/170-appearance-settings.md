# 170 외형 설정: 폰트 크기와 불투명도

## 상태

- 상태: 완료
- 브랜치: `feat/appearance-settings`

## 목표

달력 텍스트 크기와 배경 불투명도를 조절할 수 있게 해 다양한 해상도와
배경화면에서 가독성을 확보한다.

## 의존성

- 140 (설정 저장소), 150 (설정 창 배치)

## 범위

- 폰트 크기 3단계(소/중/대 — 배율 0.9/1.0/1.15)를 설정 창 "외형" 섹션에
  추가한다.
- `Themes/`의 리소스에 `DynamicResource` 기반 크기 토큰을 도입하고 배율을
  곱해 갱신한다. 하드코딩 크기의 토큰 치환 범위는 달력 셀과 헤더로
  한정한다.
- 셀·프레임 배경 불투명도 슬라이더를 추가한다.
- 변경은 즉시 반영하고 설정 저장소에 저장한다.

## 제외 범위

- 라이트/다크 테마 시스템. 현재 SOCAR 스타일 커스텀 디자인과 충돌하고
  범위가 지나치게 커진다.
- 폰트 패밀리 선택, 색상 팔레트 편집.

## 설계 결정

- 크기는 자유 값이 아닌 3단계 프리셋으로 제한해 레이아웃 검증 범위를
  고정한다.
- 불투명도는 배경 브러시에만 적용하고 텍스트에는 적용하지 않는다.
- 앱 설정 스키마를 3으로 올리고 기존 문서는 보통 크기와 90% 불투명도를 기본값으로
  정규화한다. 유효하지 않은 값은 지원 범위로 복구한다.
- 글자 프리셋은 폰트뿐 아니라 셀 헤더와 일정 행 높이 토큰도 함께 배율 적용해 큰
  프리셋에서 배지와 숫자가 잘리지 않게 한다.

## 완료 조건

- 세 단계 모두에서 셀 배지와 숫자가 잘리지 않는다.
- 설정이 재시작 후 유지된다.

## 검증

- 설정 왕복 저장 회귀 테스트.
- 지원하는 최소 창 크기에서 세 단계 화면을 수동 확인하고 PR에 스크린샷을
  포함한다.

### 실행 결과

- `dotnet build HmDesktopCalendar.csproj -c Debug`: 성공(경고 0, 오류 0).
- `dotnet run --project tests/HmDesktopCalendar.RegressionTests/HmDesktopCalendar.RegressionTests.csproj -c Debug`:
  61개 회귀 테스트 성공.
- Windows UI 자동화로 설정 창의 세 단계 글자 크기 선택기, 50~100% 불투명도
  슬라이더, 세로 스크롤과 접근성 요소를 확인했다.
- 스크린샷: `docs/screenshots/170/170-appearance-settings.png`.
- 바탕화면 레이어에 연결된 달력 본체는 자동화 캡처 대상에서 안정적으로 유지되지
  않아 최소 크기 본체 스크린샷은 만들지 못했다. 세 프리셋의 글자·셀 헤더·일정 행
  토큰 값과 불투명도 범위는 회귀 테스트로 검증했다.
