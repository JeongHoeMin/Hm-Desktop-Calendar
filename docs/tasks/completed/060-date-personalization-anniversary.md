# 060 날짜 셀 개인화와 기념일

## 상태
- 상태: 완료
- 브랜치: `feat/date-personalization-anniversary`

## 목표
날짜 셀 배경색을 계정 간 동기화하고 완료 개념이 없는 연간 기념일을 명확하게
표시·편집할 수 있게 한다.

## 의존성
- 작업 030, 040, 050 완료

## 범위
- 날짜 편집 화면에서 셀 배경 팔레트, 사용자 HEX와 초기화를 제공한다.
- 날짜 셀 전경색을 배경 명도에 따라 자동 선택한다.
- 기념일 유형, 배지와 연간 반복 편집을 제공한다.
- 2월 29일 기념일은 윤년에만 표시·알림 대상으로 만든다.

## 제외 범위
- 기념일 외 고급 일정 분류와 공유 캘린더는 지원하지 않는다.
- 날짜 셀 색상을 일정 대표 색상으로 자동 결정하지 않는다.

## 설계 결정
- 날짜 셀 배경은 일정과 독립적인 날짜별 동기화 데이터다.
- 기념일은 완료 체크가 없고 연간 반복 규칙을 자동으로 사용한다.
- 날짜 셀 배경은 기존 `DateCellDecorationKind.Highlight`로 저장하고 날짜로부터
  결정적인 ID를 만들어 여러 환경의 동시 설정도 같은 동기화 엔터티로 수렴시킨다.
- 사용자 배경색은 `#RRGGBB`로 정규화하고 짙은 차콜과 흰색 중 대비가 더 높은
  색을 날짜 셀 전경색으로 자동 선택한다.
- 기념일 전환은 시작 날짜를 기준으로 종료 없는 1년 간격 반복 규칙을 적용하며
  완료 상태를 항상 해제한다.

## 완료 조건
- 날짜 셀 색상 설정·초기화가 로컬과 계정 동기화 후 동일하게 표시된다.
- 기념일이 일반 할 일과 구분되고 윤년 정책을 따른다.

## 검증
- 색상 직렬화·동기화·전경 대비와 윤년 발생 테스트를 추가한다.
- 달력·편집 화면을 밝고 어두운 사용자 색상으로 수동 검토한다.

- .NET Release 빌드를 경고 0개, 오류 0개로 통과했다.
- 날짜 배경 정규화·결정적 ID·전경 대비, 기념일 연간 반복과 편집 흐름을 포함한
  회귀 테스트 39개를 통과했다.
- 서버 테스트 9개와 TypeScript 빌드를 통과했다.
- Avalonia Headless로 편집 창 640×560 최소 크기와 820×700 일반 크기에서
  어두운 배경·밝은 배경·기념일 편집을 확인하고, 달력 700×480에서 날짜 배경과
  기념일 배지를 확인했다.
- 검증 캡처는 `docs/screenshots/060-editor-dark-minimum.png`,
  `docs/screenshots/060-editor-light.png`,
  `docs/screenshots/060-main-background-anniversary.png`에 기록했다.
