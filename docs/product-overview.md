# Hm Desktop Calendar 제품 개요

## 목적

Hm Desktop Calendar는 Windows 바탕화면에 월간 달력과 날짜별 할 일을 자연스럽게
배치하는 데스크톱 애플리케이션이다. 사용자가 별도 웹페이지나 일정 앱을 열지
않아도 바탕화면에서 할 일을 확인하고 빠르게 편집할 수 있게 하는 것이 핵심
목적이다.

제품은 다음 경험을 우선한다.

- 달력이 일반 앱 창처럼 작업을 방해하지 않고 바탕화면의 일부처럼 동작한다.
- 로그인하지 않아도 로컬에서 즉시 사용할 수 있다.
- 로그인한 사용자는 여러 실행 환경 사이에서 변경 사항을 동기화할 수 있다.
- 일정 데이터와 창 배치가 앱을 다시 시작해도 유지된다.
- Windows 환경에서 예측 가능하고 복구 가능한 방식으로 동작한다.

## 현재 제공 기능

### 바탕화면 달력

- Windows Explorer의 바탕화면 레이어에 월간 달력 창을 연결한다.
- 바탕화면 아이콘을 달력보다 위에 유지한다.
- 이전 달과 다음 달로 이동할 수 있다.
- 상단의 연도 목록과 12개월 선택기에서 표시 월을 바로 이동하고 오늘로 돌아갈 수
  있다.
- 로그인 여부와 동기화 진행·실패·마지막 성공 시각을 상단에서 확인할 수 있다.
- 날짜 셀에 미완료·완료 할 일 수와 표시 가능한 할 일 일부를 보여준다.
- 일정마다 저장한 접근 가능한 텍스트 색상을 날짜 셀 미리보기에 반영한다.
- 날짜별 사용자 배경색을 팔레트나 HEX로 설정하고 밝기에 따라 날짜 전경색을
  자동으로 선택하며, 이 설정을 계정 간 동기화한다.
- 완료 상태가 없는 연간 기념일을 일정과 구분된 배지로 표시하고 2월 29일
  기념일은 윤년에만 발생시킨다.
- 한국 법정 공휴일과 대체공휴일을 로컬에서 계산해 날짜 숫자와 셀 헤더 배지로
  표시하며, 같은 날 겹친 공휴일 이름을 함께 보여준다.
- 설정에서 주 시작 요일을 일요일 또는 월요일로 바꾸고, 일요일·토요일 날짜 숫자
  색상을 켜거나 끌 수 있다. 사용자 배경색과 공휴일 색은 주말 색보다 우선한다.
- 달력 헤더·셀 글자 크기를 작게·보통·크게 조절하고, 텍스트에는 영향을 주지 않는
  프레임·날짜 셀 배경 불투명도를 50~100% 범위에서 설정할 수 있다.
- 표시 공간을 넘는 할 일은 숨겨진 개수로 요약한다.
- 바탕화면의 날짜 셀을 더블클릭하면 해당 날짜 편집 창을 연다.
- 다른 앱 창, 바탕화면 아이콘 또는 유효하지 않은 창을 더블클릭하면 편집 창을
  열지 않는다.

### 할 일 관리

- 날짜별 할 일을 추가, 수정, 완료 및 삭제할 수 있다.
- 제목, 선택적 시각과 메모를 저장할 수 있다.
- 시각이 없는 할 일과 자주 사용하는 시각 프리셋을 지원한다.
- 추가와 수정에 같은 편집 폼을 사용하고 메모를 주요 입력 필드로 제공한다.
- 중립 일정 칩에서 WCAG AA 대비를 만족하는 팔레트와 사용자 지정 HEX 텍스트
  색상을 지원한다.
- 하루 일정, 연속 기간 일정과 매일·매주·매월·매년 반복 일정을 같은 편집 폼에서
  만들고 종료일, 간격, 주간 요일과 선택적인 반복 종료일을 설정할 수 있다.
- 기간이나 반복 일정의 파생 날짜에서 편집해도 원본 전체 시리즈를 수정·완료·삭제하며
  목록 배지와 편집 안내에서 적용 범위를 확인할 수 있다.
- 저장하지 않은 변경을 버리거나 일정을 삭제하기 전에 사용자 확인을 받는다.
- 동일 날짜의 할 일을 완료 상태, 시각, 제목 순으로 정렬한다.

### 일정 모아보기

- 메뉴에서 별도 일정 모아보기 창을 열어 현재 월, 오늘부터 30일·90일 또는 사용자 기간의
  발생 일정을 조회한다.
- 제목과 메모를 검색하고 예정·완료·기념일 상태로 필터링하며 날짜와 시간의 빠른 순 또는
  늦은 순으로 정렬한다.
- 반복 일정은 선택 기간 안의 발생 항목만 표시하고, 목록에서 기존 공용 편집 창을 열어
  전체 시리즈 원본을 추가·수정·완료·삭제한다.
- 로컬 저장과 원격 동기화 변경이 연속되면 목록 조회를 디바운스해 최신 상태를 반영한다.
- 전체 일정 시리즈를 표준 iCalendar(.ics) 파일로 원자적으로 내보내 Outlook 등 외부
  캘린더에서 열 수 있다.

### 일정 알림

- 일정 시작 시각을 기준으로 정시·5·15·30분·1시간·1일 전 또는 사용자 지정 분에
  앱 자체 알림 팝업을 표시한다.
- 시간이 없는 일정은 사용자가 별도 알림 시각을 지정하며, 기간 일정은 시작일에
  한 번, 반복 일정과 기념일은 각 발생 날짜마다 알린다.
- 팝업에서 확인하거나 5·10·30분 뒤로 미루고 해당 일정 편집 창을 열 수 있다.
- 동시에 도래한 알림은 창 하나에서 순서대로 표시하고, 앱 시작 시 최근 24시간
  안에 놓친 알림만 장치별로 한 번 복구한다.
- 알림 설정은 계정 간 동기화하지만 표시·확인·미루기 기록은 각 장치에만 저장한다.

### 로컬 저장과 계정 동기화

- 앱 시작 후 루트 일정 JSON과 계정별 `accounts` 데이터를 하루 한 번 로컬 백업하고
  최근 10개 세대를 유지한다. 설정에서 마지막 백업 시각과 백업 폴더를 확인한다.
- 로그인하지 않은 상태에서도 JSON 기반 로컬 저장소를 사용한다.
- 버전 2 캘린더 문서는 일정 시리즈, 날짜 셀 장식과 통합 동기화 커서를 함께
  저장하며 최초 생성 시 기존 `todos.json`을 원자적으로 가져온다.
- 데스크톱 달력과 편집기는 버전 2 일정 저장소와 발생 일정 조회를 사용한다.
- 이메일 계정 등록, 로그인, 로그아웃과 세션 복원을 지원한다.
- 로그인 계정별 로컬 저장 영역을 분리한다.
- PostgreSQL 기반 서버와 변경 사항을 양방향 동기화한다.
- 서버 변경 알림을 WebSocket으로 받아 동기화를 다시 요청한다.
- 삭제 항목과 서버 커서를 사용해 여러 변경을 순서대로 반영한다.
- 네트워크를 사용할 수 없는 동안 로컬 변경을 유지하고 이후 동기화를 재시도한다.

### 창과 애플리케이션 수명주기

- 트레이와 달력 메뉴에서 단일 설정 창을 열고 앱 버전을 확인하거나 달력 창 위치를
  기본값으로 초기화할 수 있다.
- 트레이 메뉴에서 설정, 일정 모아보기, 달력 위치 편집, 로그인·로그아웃과 종료를
  실행할 수 있으며 로그인 상태에 따라 메뉴 문구를 갱신한다.
- 설정에서 현재 사용자 Windows 로그인 시 앱 자동 실행을 등록하거나 해제할 수
  있으며 레지스트리 접근 실패 시 토글과 오류 사유를 함께 표시한다.
- 메뉴에서 달력의 위치와 크기를 수정, 저장 또는 취소할 수 있다.
- 다중 모니터와 음수 화면 좌표를 고려해 달력 위치를 유지한다.
- Windows Explorer가 다시 시작되면 바탕화면 레이어 연결을 복구한다.
- 종료 시 알림 예약, 마우스 훅, 실시간 연결, 동기화 작업과 저장소를 순서대로
  정리한다.

## 시스템 구성

### 데스크톱 클라이언트

- .NET과 Avalonia UI로 구현된 Windows 데스크톱 애플리케이션이다.
- MVVM 기반 화면 모델과 로컬 우선 저장소를 사용한다.
- Win32 API를 통해 바탕화면 레이어, 전역 포인터 입력, 창 위치와 모니터 정보를
  처리한다.

### 동기화 서버

- Node.js, TypeScript와 Fastify로 구현된다.
- PostgreSQL에 사용자, 갱신 세션, 기존 할 일과 버전 2 일정·날짜 장식 데이터를
  저장한다.
- JWT 접근 토큰과 회전 가능한 갱신 토큰으로 인증한다.
- HTTP API는 기존 버전 1 계약과 일정·날짜 장식용 버전 2 통합 커서 동기화를
  함께 제공하고, WebSocket은 변경 발생을 클라이언트에 알린다.

## 제품 경계

- 주 실행 환경은 Windows이며 Win32 바탕화면 통합은 핵심 기능이다.
- 로컬 사용은 서버 없이 가능해야 한다.
- 원격 동기화는 서버와 PostgreSQL이 준비된 경우에만 활성화된다.
- 바탕화면 달력은 다른 앱 위에 영구적으로 표시되는 위젯이 아니다.
- 이 문서는 현재 구현과 장기적으로 유지되는 제품 목적만 설명한다. 작업별 요구,
  설계 결정, 진행 상태와 PR 정보는 `docs/tasks`에서 관리한다.

## 문서 유지 규칙

- 현재 기능이나 시스템 경계가 바뀌면 해당 구현 PR에서 이 문서를 함께 갱신한다.
- 아직 구현되지 않은 기능을 현재 제공 기능으로 기록하지 않는다.
- 개별 작업의 임시 계획이나 상태를 이 문서에 추가하지 않는다.
